# スタッフ権限のセットアップ

## 1. マイグレーションの実行（初回のみ）

Supabase Dashboard の **SQL Editor** で以下を実行してください。

1. https://supabase.com/dashboard にログイン
2. 対象プロジェクトを選択
3. 左メニュー → **SQL Editor** → **New query**
4. `supabase/migrations/008_user_roles.sql` の内容をコピーして貼り付け
5. **Run** をクリック

※ 既存テーブル・既存データには一切触れません。新規テーブル `user_roles` のみ作成されます。

## 2. ロールの設定

### 方法A: 管理画面から（推奨）

1. 管理者でログイン
2. サイドバー → **ユーザー管理**
3. 「ユーザーを追加」で新規ユーザー作成（メール・パスワード・権限を指定）
4. 既存ユーザーの「管理者」「スタッフ」ボタンで権限を切り替え

### 方法B: スクリプトから

```bash
# 管理者（全機能）にする場合
node scripts/set-user-role.mjs "メールアドレス" admin

# スタッフ（打刻・シフトのみ）にする場合
node scripts/set-user-role.mjs "メールアドレス" staff
```

## 3. 権限の違い

| 権限 | 管理者 | スタッフ |
|------|--------|----------|
| 打刻 | ○ | ○ |
| シフト | ○ | ○ |
| 勤怠管理 | ○ | × |
| スタッフ管理 | ○ | × |
| 店舗管理 | ○ | × |
| ポリシー | ○ | × |
| データ出力 | ○ | × |

## 4. 補足

- **user_roles にレコードがないユーザー** → 管理者として扱われます（後方互換）
- スタッフが管理者専用ページに直接 URL でアクセスした場合 → 自動でシフト画面にリダイレクトされます
